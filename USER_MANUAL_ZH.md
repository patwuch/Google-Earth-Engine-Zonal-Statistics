# 第一部分 — 使用說明

## 1. 概述

GEE Web App 讓您從 Google Earth Engine 擷取任意地理範圍的衛星資料。您需要定義：

- **位置** — 上傳邊界檔案（研究範圍）
- **產品** — 選擇一個或多個資料集及其變數
- **時段** — 為每個資料集設定日期範圍

應用程式會在背景將請求送往 Google Earth Engine，完成後產生可下載的檔案（GeoParquet 與 CSV）。依據範圍大小和日期跨度，執行時間從數分鐘到數小時不等。

---

## 2. 啟動與停止應用程式

依您選擇的執行方式，閱讀對應的區塊即可。

---

### 使用 Docker（僅限 Linux）

> Docker 目前尚未支援 Windows。Windows 使用者請使用下方的 Pixi 區塊。

**啟動：** 在資料夾中開啟終端機並執行：

```
./docker.sh start
```

**停止：**

```
./docker.sh stop
```

---

### 使用 Pixi（Windows 與 Linux）

**啟動：**

| 平台 | 操作方式 |
|------|----------|
| Windows | 雙擊 `pixi.bat`，選擇 **start** |
| Linux | 在終端機中執行 `./pixi.sh start` |

**停止：**

| 平台 | 操作方式 |
|------|----------|
| Windows | 雙擊 `pixi.bat`，選擇 **stop** |
| Linux | 在終端機中執行 `./pixi.sh stop` |

---

首次啟動需要下載並安裝所有依賴套件（3–10 分鐘）。之後每次啟動都會快得多。準備就緒後，瀏覽器將自動開啟。

關閉瀏覽器分頁**不會**停止應用程式。任何正在進行的擷取作業將持續在背景執行，直到您使用停止指令或作業完成為止。

---

## 3. 上傳 GEE 金鑰

第一次開啟應用程式時，畫面會顯示上傳提示而非主介面。將您的 `.json` 金鑰檔案拖曳到上傳區域，或點擊瀏覽檔案。應用程式會驗證金鑰，成功後顯示服務帳戶的電子郵件地址。

金鑰會儲存在本機，每次重新啟動應用程式時自動使用。除非您更換金鑰，否則不會再次詢問。

**更換金鑰：** 在左側邊欄頂部找到 **GEE Credentials** 區塊，其中顯示已連線的帳戶電子郵件及 **Replace** 按鈕。點擊即可上傳不同的金鑰。

---

## 4. 管理執行作業

每次擷取作業會組織成一個**執行（run）**——一筆包含設定、輸入幾何、日誌與輸出檔案的獨立記錄。

### 建立新的執行

點擊側邊欄 **Run Session** 區塊中的 **+ New** 按鈕。應用程式會預先填入唯一的執行 ID；您可以保留或自行輸入。點擊 **Confirm** 建立執行。

### 選取先前的執行

開啟側邊欄中的 **Previous runs** 下拉選單。每一列顯示執行 ID、狀態（running、completed、failed、stopped）及所包含的資料集。點擊任一列即可載入該執行，查看完整狀態、日誌與結果。

### 執行 ID

執行 ID 是將該次執行所有檔案串連在一起的短識別碼。您可以在 `data/runs/<run_id>/results/` 中找到輸出檔案。

---

## 5. 上傳研究範圍

選取執行後，左側主要區域會顯示**研究範圍（Area of Interest）**上傳區。將邊界檔案拖曳至此，或點擊瀏覽。接受的格式：

| 格式 | 副檔名 |
|------|--------|
| Shapefile | 包含 `.shp`、`.shx`、`.dbf`、`.prj` 的 `.zip` |
| GeoJSON | `.geojson` |
| GeoParquet | `.parquet` |

上傳後，應用程式會顯示幾何圖形的地圖預覽（綠色），以及要素數量、座標系統與範圍框。

提交執行後，幾何圖形即鎖定於該執行，無法更改。若載入先前的執行，幾何圖形已儲存，無需重新上傳。

---

## 6. 設定資料集

側邊欄的 **Datasets** 區塊列出所有可用產品，預設為收合狀態。

**啟用資料集：**

1. 勾選核取方塊以啟用，設定卡片隨即展開。
2. 設定**日期範圍**（開始與結束日期）。每個資料集的有效日期會自動限制。
3. 選擇要擷取的**波段**。預設全選——取消勾選不需要的波段即可。
4. 選擇要計算的**統計量**（例如 mean、min、max、median）。適當的預設值已預先填入。

一次執行可啟用多個資料集，系統會並行處理。

**可用資料集：**

| 資料集 | 量測內容 | 解析度 | 日期範圍 |
|--------|----------|--------|----------|
| **CHIRPS** | 降水量 | ~5.6 km | 1981 → 至今 |
| **ERA5-Land** | 氣溫、降水量、蒸發量（11 個變數） | ~9 km | 1950 → 至今 |
| **MODIS LST** | 地表溫度（白天 + 夜間） | 1 km | 2000 年 2 月 → 至今 |
| **MODIS NDVI/EVI** | 植被指數 | 250 m | 2000 年 2 月 → 至今 |
| **NDBI** | 歸一化建築指數（Landsat 5 / 7 / 8） | 30 m | 1984 → 2025 |
| **WorldCover v1.0** | 土地覆蓋分類 | 10 m | 僅 2020 年 |
| **WorldCover v2.0** | 土地覆蓋分類 | 10 m | 僅 2021 年 |
| **MODIS LULC** | 土地利用與土地覆蓋類型 | 500 m | 2001–2023 |

---

## 7. 執行分析

上傳研究範圍並設定至少一個資料集（已選擇波段與統計量）後，右側面板的 **Run Analysis** 按鈕會變為可點擊狀態。點擊以提交。

應用程式會凍結設定、記錄開始事件，並在背景開始從 Google Earth Engine 擷取資料。您可以安心關閉瀏覽器分頁——擷取作業將持續在背景進行，回來後結果將等候您下載。

---

## 8. 監控進度

執行進行中，您可以在右側面板追蹤進度：

**進度條** — 顯示已完成的分塊（子任務）數量佔總數的比例。大範圍和長日期跨度會被切割成許多較小的分塊；此進度條讓您了解執行進展。失敗的分塊以紅色顯示。

**Snakemake Log** — 顯示於左側地圖下方，為擷取管線的即時輸出，以顏色區分：綠色為已完成步驟、紅色為錯誤、黃色為警告。每隔幾秒更新一次。

**Run Event** — 位於側邊欄底部，顯示所有執行的帶時間戳訊息。錯誤以紅色顯示。

---

## 9. 下載結果

執行完成（狀態：**completed**）後，右側面板會出現 **Download Results** 區塊。每個資料集產品都有兩個按鈕：

- **GeoParquet** — 包含幾何資訊的完整結果檔案，可在 Python（GeoPandas）、QGIS 或類似工具中使用。
- **CSV** — 相同資料的純表格版本（不含幾何）。適合試算表工具使用。

兩個檔案也會直接儲存至您的電腦：

```
data/runs/<run_id>/results/<product>_<start>_to_<end>.parquet
data/runs/<run_id>/results/<product>_<start>_to_<end>.csv
```

---

## 10. 部分下載（執行中途下載）

若執行尚未完成就需要結果——例如，執行仍在處理較晚的年份，但您現在想要早期的結果——請使用**部分下載（partial checkout）**。

1. 選取執行後，在右側面板找到 **Build Partial Checkout** 按鈕。
2. 點擊它。應用程式會將目前已完成的所有分塊合併為每個產品的單一檔案。
3. 下載按鈕將顯示 **GeoParquet** 和 **CSV**，與完成的執行相同。

隨時再次點擊 **Build Partial Checkout** 以納入更多已完成的分塊。

---

## 11. 停止、重試與恢復執行

### 停止執行中的作業

在執行進行時，點擊右側面板的 **Stop Run** 按鈕（紅色）。管線將停止，執行狀態標記為 **stopped**。

### 重試失敗或已停止的執行

若執行顯示狀態 **failed** 或 **stopped**，會出現 **Retry Run** 按鈕。點擊後將從中斷處繼續管線——已完成的分塊不會重新擷取。

### 從先前的工作階段恢復執行

從側邊欄的 **Previous runs** 下拉選單選取該執行。執行狀態、日誌及可用結果將自動載入。若上次關閉應用程式時執行仍在進行，其狀態將反映背景執行的結果。

# 第二部分 — 技術參考

## 12. 系統需求

您需要以下**其中一種**執行環境，以及一個 GEE 金鑰。

### 選項 A — Docker（僅限 Linux）

適合：完整的環境隔離。Docker 在容器中執行所有程序，除 Docker 本身外不在您的電腦上安裝任何東西。

> **Docker 目前尚未支援 Windows。** Windows 使用者必須使用 Pixi 選項。

安裝 [Docker Desktop](https://www.docker.com/products/docker-desktop) 並確保在啟動應用程式前已開啟並執行。無需其他軟體。

### 選項 B — Pixi（Windows 與 Linux）

適合：Docker 受限或無法使用的環境（例如企業管理電腦、大學網路，或任何無法安裝 Docker Desktop 的地方）。目前也是 Windows 使用者的唯一選項。

Pixi 是一款輕量套件管理器，直接在您的電腦上安裝應用程式依賴套件，無需容器。

**若尚未安裝 Pixi，系統會自動安裝。** 在 Windows 與 Linux 上，若未偵測到 Pixi，啟動腳本會提示您在繼續前安裝。無需手動操作。

### GEE 服務帳戶金鑰

一個 `.json` 檔案，用於授權應用程式存取 Google Earth Engine。若您尚未取得，請參閱 README 中的[取得金鑰](README.md#getting-a-key)說明。

## 13. 疑難排解

### Docker

| 症狀 | 可能原因 | 處理方式 |
|------|----------|----------|
| 啟動後瀏覽器未開啟 | Docker 未執行 | 開啟 Docker Desktop 並等待啟動完成，然後再試 |
| Linux 上出現檔案所有權錯誤 | UID/GID 不符 | 請務必使用 `./docker.sh start` 啟動應用程式，而非手動啟動 Docker |

### Pixi

| 症狀 | 可能原因 | 處理方式 |
|------|----------|----------|
| 啟動後瀏覽器未開啟 | Pixi 未安裝或不在 PATH 中 | 重新執行啟動腳本並接受安裝提示；若仍失敗，開啟新終端機再試一次 |
| 安裝提示後找不到 `pixi` 指令 | 目前工作階段的 PATH 尚未更新 | 關閉並重新開啟終端機，然後再次執行 `./pixi.sh start` |

### 一般問題

| 症狀 | 可能原因 | 處理方式 |
|------|----------|----------|
| 應用程式開啟後立即顯示上傳提示 | 尚未儲存 GEE 金鑰 | 上傳您的 `.json` 服務帳戶金鑰 |
| 上傳金鑰後出現「Authentication error」 | 金鑰無效或服務帳戶缺少 Earth Engine 存取權限 | 確認服務帳戶在 Google Cloud Console 中具有 **Earth Engine** 角色 |
| 執行立即顯示 **failed** | 設定或幾何圖形問題 | 查看事件日誌中的錯誤訊息；修正問題後點擊 **Retry Run** |
| 進度條長時間卡住 | GEE 速率限制或網路問題 | 等待幾分鐘；查看 Snakemake Log 中的錯誤行 |
| 點擊下載按鈕沒有反應 | 結果檔案尚未產生 | 確認執行已完成，或對進行中的執行使用 **Build Partial Checkout** |
| 關閉瀏覽器後應用程式停止回應 | 預期行為——應用程式仍在執行 | 重新開啟瀏覽器，並前往啟動時顯示的網址 |
